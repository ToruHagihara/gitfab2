「ものづくり」と「ものがたり」の統合プラットフォーム　“GitFAB”

Making stories of making something

概要

GitFABは、「文脈やストーリーと深く紐づいたモノづくり(FAB)」プロジェクトのための、共有ウェブサービスです。

主にソフトウェア開発プロジェクトのための共有ウェブサービスである「GitHub」、ソーシャルネットワーキングサービスである「Facebook」とも部分的に連動し、モノづくりのプロジェクト全体の進行を管理、さらに、モノづくりの知識を適切に記述・保存・蓄積・伝達していくことを支援します。

機能

GitFABは、モノづくり(FAB)プロジェクトを、「Recipe(レシピ)」と「Notel(ノート)」という２つの相互補完的な形式で記述していくことができます。「Note(ノート)」は、自然言語による自由記述方式です。

本システムの特徴である「Recipe(レシピ)」は、複数の写真や映像を、「有向グラフ（矢印）」を用いて関連付け、「制作過程」や「状態遷移」をまとめていくことで、非言語的な情報伝達を実現するものです。この「Recipe(レシピ)」は、モノづくりにおける、再現可能で分かりやすい形式であるとの仮説のもとに、採用したものです。「Recipe(レシピ)」には、適宜、短いコメントやアノテーションをつけることもできます。

さらに、GitFABでは、「スライドショー」機能を設けています。「スライドショー」中は、「Recipe(レシピ)」に投稿された写真や映像が、有向グラフ（矢印）で結ばれた順番にスクリーンに拡大表示されます。このため、別途プレゼンテーション資料をつくらなくても、GitFAB上で直接プレゼンテーションを行うことができます。

特徴

GitFABが、他の同種のサービスと最も異なる点は、モノづくり(FAB)プロジェクトの「はじまり（初期段階）」から「おわり（最終段階）」まで、あらゆる段階を支援するため、日常的に使うことができることです。

モノづくり(FAB)プロジェクトの「はじまり（初期段階）」では、制作者本人が、自分自身のために「制作ノート」「制作メモ」や「試行錯誤の過程」を整理しながらつけていくことができます。

モノづくり(FAB)プロジェクトの「おわり（最終段階）」では、他の人も同じつくりかたを再現できるように、必要最小限の情報だけに絞った、まとまった「手順」を保管することができます。

また、ある制作者にとってプロジェクトが「おわり（最終段階）」であっても、それを見た別の制作者が、そのプロジェクトを引用することで、また別のプロジェクトを「はじめる（初期段階）」ことができます。GitHubと同様、GitFabでは、こうした「フォーク（分岐）」の機能を重要視しています。

さらに、モノがつくられた後にでも、その「使い方」や、そこでの気づきや発見を蓄積していくことが可能です。

このようなさまざまな機能を有し、さらに「グループ機能」を持っているため、個人のみならずグループ単位のプロジェクト管理にも有効です。メイン開発者、サブ開発者、プロジェクトパートナー、感想コメント、アドバイス、試用感など、さまざまな参加の方式、コミットメントの度合いを許容することも大きな特徴です。

GitHubとの連動

gitFAB へのログインは GitHub 認証で行い、プロジェクトデータも GitHub 上に保存されていきます。データのバックアップという意味もありますが、GitHub コミュニティに対して活動を共有できるため、新たなプロジェクト貢献者の参加を促すことが期待できます。

Facebookとの連動

プロジェクトや日誌などに対するフィードバックは Facebook コメントを利用します。また、プロジェクトの追加や更新などは Facebook ページに連動させる予定です。ソーシャルへの露出を高め、気軽なコメントや感想を多く受け取ることができます。
